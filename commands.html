Every Command Of Aucbot

Behold, a simple walkthrough detailing each command of the Aucbot awaits you.

This is the command by which you can set the purse of a user.

Required Arguments: Purse can be in million or billions. For your ease, I have added 'm' and 'b' features. For example, if you want to set the purse of user @Khawer#6683 to 3 million, you can type [prefix] setpurse @Khawer#6683 3m. This will set their purse to 3 million. If you use the command prefix setpurse @Khawer#6683 3 millions, it will break.

Aliases: sp

Required Permission/Role: You need administrator permission or the Auctioneer (case sensitive) role to be able to run this command.

This is the command by which you can set the logging channel where the bot will type all the logs of your auction.

Required Arguments: You need to provide a logging channel or its ID.

Aliases: slc

Required Permission/Role: You need administrator permission or the Auctioneer (case sensitive) role to be able to run this command.

This command allows you to check your or anyone's purse.

Required Arguments: [member] (this is an optional argument). If you want to check someone else's purse, you can mention their name or username. If not, the command will display your purse.

Aliases: b, wallet

Required Permission/Role: None

This is the command to start an auction on a player.

Required Arguments: <player you want to sell> <base price> [increment]. For example, $sell Kohli 4m 3m. Note that this command has a simple problem where it may break if you sell a player who has two names, such as Babar Azam. To avoid this problem, it's advised to use the slash command /sell instead.

Aliases: None

Required Permission/Role: You need administrator permission or the Auctioneer (case sensitive) role to be able to run this command.

This command allows you to delete a player from someone's squad or even delete all players.

Required Arguments: The 'index' can be set to 'all', 'everyone', or 'max' to delete all players. If you want to delete a specific player, you can check that player's index by using the 'squad' command and set the 'index' to that specific index. This command requires administrator permissions to run and can be used to delete players from a user's squad or delete all players.

This is a premium command that requires users to invite 5 other users to the support server of Aucbot.

Command: /set_automatic_sold_time [time]

Required Argument: [time] - The amount of time in seconds to set as the time limit for automatic sold. The maximum value that can be set is 30 seconds.

Example: /set_automatic_sold_time 15 - This will set the time limit to 15 seconds.

To gain access to this premium command, users are required to invite 5 other users to the support server of Aucbot. Once the invites are confirmed, they will be able to use this command to configure the time limit for automatic player selling.

The swap command allows you to change the position of players within your squad.

Command: .swap <index1> <index2>

Required Permission: None

Required Arguments: You need to specify the indices of the players you want to swap. Use <index1> to indicate the index of the first player and <index2> to indicate the index of the second player.

Example: .swap 2 5 - This will swap the players at index 2 and index 5 within your squad.

This command is easy to use and doesn't require any special permissions. It allows you to rearrange the positions of players in your squad as needed.

The 'swappurse' command allows you to swap the purse values between two members. For example, if your purse is 6 and another member's purse is 9, this command will switch the purse values between the two members.

Command: swappurse @member1 @member2

Required Permission/Role: You need administrator permission or the Auctioneer (case sensitive) role to use this command.

Example: swappurse @khawer @zuhair - This will swap the purses of the members @khawer and @zuhair.

By using this command, you can exchange the purse values between members, facilitating transfers or adjustments within the auction system. Please note that only users with the necessary permissions or role can execute this command.

The 'swapsquads' command allows you to swap the squads between two members. This command will exchange the squads of the specified members.

Command: swapsquads @member1 @member2

Required Permission/Role: You need administrator permission or the Auctioneer (case sensitive) role to use this command.

Example: swapsquads @khawer @zuhair - This will swap the squads between the members @khawer and @zuhair.

Using this command, you can exchange the squads between members, facilitating team transfers or adjustments within the auction system. Please note that only users with the necessary permissions or role can execute this command.

The 'prefix' command allows you to change the default prefix used to interact with the bot. By default, the prefix is set as '$', but you can modify it using this command.

Command: prefix [new_prefix]

Required Permission/Role: You need administrator permission to change the prefix.

Example: $prefix . - This will change the prefix to '.'

After changing the prefix, you can use the new prefix followed by the command name or by mentioning the bot to execute commands. For example, if the new prefix is '.', you can use commands like '.help' or mention the bot and say 'help' to trigger the help command.

Remember to have administrator permission to change the prefix and ensure to choose a prefix that does not conflict with other existing commands or bots in your server.

The "export_data" command allows server administrators with the "manage guild" permission to generate a comprehensive txt file. This file contains detailed rosters and corresponding purses for all members in the server, providing valuable information in a user-friendly format.

The `delete_server_data` command enables server administrators to permanently remove all server-related data, including player rosters and purses. It should be used with extreme caution, as the action is irreversible. Before executing the command, a confirmation prompt will be displayed to ensure that only authorized server administrators with the necessary permissions can initiate this operation. Please exercise caution while using this command to avoid unintended data loss.
